# Demo Script — AI-Driven Emotion Detection & Personalized Learning Support Platform

A step-by-step script for opening and demonstrating the app to mentors.

---

## Before You Start (Setup Checklist)

1. Open a terminal and navigate to the project folder:
   ```
   cd C:\ai-project
   ```
2. Activate your environment (if using one) and make sure dependencies are installed:
   ```
   pip install -r requirements.txt
   ```
3. Make sure your `.env` file has a valid `GEMINI_API_KEY` (copy from `.env.example` if not done already).
4. Launch the app:
   ```
   streamlit run app.py
   ```
5. It should open automatically in your browser at `http://localhost:8501`. If not, open that link manually.

---

## Opening Line (say this first)

> "This is an AI-Driven Emotion Detection & Personalized Learning Support Platform. It's built to solve a simple problem — students often get stuck, confused, or frustrated while studying, but there's no one around at 1am to notice that and help. This app detects a student's emotional state from what they type, and responds with personalized, encouraging support in real time."

---

## Part 1: Introduce the Problem (30 seconds)

> "Most learning platforms only track whether an answer is right or wrong — they don't track *how the student feels* while getting there. A student who's silently anxious or about to give up looks the same to the system as one who's doing fine. That's the gap this project addresses."

---

## Part 2: Live Demo — Check In Page

1. You'll land on the **"Check In"** page by default.
2. Point to the text box and say:
   > "Here, a student types how they're feeling about their studies — in their own words, no forms or dropdowns."
3. Type a sample input, e.g.:
   ```
   I'm really confused about this topic and feel like giving up
   ```
4. Click **"Analyze & Get Support"**.
5. While it loads, explain:
   > "Behind the scenes, this text goes through a hybrid emotion classifier — a BiLSTM deep learning model blended with keyword-based detection, so even if the model is uncertain, the keyword layer catches strong emotional signals."
6. Once results appear, walk through the screen:
   - **Detected emotion** — "Here's the final emotion it settled on."
   - **Model prediction vs confidence** — "This shows what the neural network predicted, and how confident it was."
   - **Keyword match vs source used** — "This shows whether the keyword layer or the model made the final call — that's the 'hybrid' part."
   - **Personalized Suggestion** — "And this is generated by the Gemini API — a short, supportive response tailored to exactly what the student said and how they're feeling, not a generic canned message."

7. Optional — do a second example live to show variety, e.g.:
   ```
   I finally understood recursion, this feels great!
   ```
   Point out how the tone of the suggestion changes completely based on emotion.

---

## Part 3: Live Demo — Analytics Dashboard

1. Click **"Analytics Dashboard"** in the sidebar.
2. Say:
   > "Every interaction gets logged automatically, so over time we can see patterns — not just for one student, but across a whole class."
3. Walk through:
   - **Total interactions logged** — quick number.
   - **Emotion Distribution chart** — "This bar chart shows which emotions come up most often — a teacher could use this to see if a class is generally struggling or doing fine."
   - **Emotion Over Time chart** — "This shows emotional trends over time, so you could catch a student sliding toward frustration before it becomes a bigger problem."
   - **Raw log data (expandable)** — "And the underlying data is always available for deeper analysis."

## Part 4: Wrap-Up / Closing Statement

> "So to summarize — this platform combines a BiLSTM emotion classifier, a keyword-based fallback for reliability, and the Gemini API for generating genuinely personalized encouragement, all logged and visualized so the support isn't just reactive for one student, but observable at scale. It was built, tested, and documented individually from end to end."

## Anticipated Mentor Questions (be ready for these)

**Q: Why BiLSTM instead of a simpler model?**
> "BiLSTM captures context from both directions in a sentence, which matters for emotion — the meaning of a phrase like 'I'm not stressed' depends on word order and negation, which a bag-of-words model would miss."

**Q: Why blend keywords with the model instead of just trusting the model?**
> "The model can be uncertain on short or ambiguous inputs. The keyword layer acts as a safety net — if confidence is low but there's a strong emotional keyword present, we trust that signal instead."

**Q: What happens if the Gemini API is down or there's no internet?**
> "There's a built-in fallback — if the API call fails or no key is set, the app still returns a supportive, pre-written message based on the detected emotion, so the app never breaks for the student."

**Q: How is data privacy handled?**
> "Interactions are logged locally to CSV in this version — no data leaves the system except the text sent to Gemini for generating the suggestion."

**Q: What would you improve with more time?**
> "Expanding the training dataset for the BiLSTM model, adding user authentication so the dashboard can show per-student history, and adding a teacher-facing view separate from the student check-in."

---

## Tips for the Actual Demo

- Test your exact demo inputs beforehand so you know what emotion/suggestion they'll produce — don't improvise live unless you're confident.
- Keep your terminal visible in a second window in case a mentor asks to see the code or logs.
- If Gemini API fails during the demo (network issues), don't panic — mention the fallback system is exactly designed for this, and it will still show a supportive message.
